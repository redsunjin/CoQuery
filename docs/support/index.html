CoQuery Support

CoQuery is a TestFlight-first SQL practice shell for iPhone and iPad. The first iOS build is intended to validate packaging, launch behavior, and the bundled Training Mode practice path.

What This Build Supports

  • Bundled SQL practice problems.
  • Sample schema inspection.
  • Read-only practice queries.
  • Local practice attempt review.
  • Training Mode boundary checks.

What This Build Does Not Support

  • Production database connections.
  • Production database credentials.
  • Hosted public web service behavior.
  • Embedded Python local server behavior.
  • Provider-backed AI feedback as a required path.

Tester Checklist

  1. Launch the app.
  2. Confirm Training Mode is visible.
  3. Run practice_list.
  4. Open a practice problem.
  5. Submit a read-only SELECT answer.
  6. Confirm production database actions are unavailable in the iOS Training shell.

Known Limits

This is an early TestFlight shell. It is not a production database client and does not embed the desktop local server.

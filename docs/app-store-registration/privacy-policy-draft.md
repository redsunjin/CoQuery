# CoQuery Privacy Policy Draft

Date: 2026-07-09
Status: Draft for a hosted privacy policy page before TestFlight/App Store submission.

Reference checked on 2026-07-09:

- Apple App Store Connect app privacy: https://developer.apple.com/help/app-store-connect/manage-app-information/manage-app-privacy/

## Overview

CoQuery is a SQL practice app for iPhone and iPad. The first TestFlight build is Training Mode only and uses bundled sample data.

## Data Collection

The first iOS Training build does not collect personal data from the app.

It does not require an account. It does not ask for production database credentials. It does not require an API key. It does not include advertising or tracking SDKs.

## Local Device Data

The app may store practice attempts and UI preferences locally on the device so testers can review their own practice flow. This local data is not sent to CoQuery by the first iOS Training build.

## Production Databases

The first iOS Training build is not a production database client. It does not connect to production databases and does not request production database credentials.

## AI Providers

The first iOS Training build does not require provider-backed AI feedback. If a future build adds provider-backed features, this privacy policy and App Store privacy responses must be reviewed before submission.

## Contact

Support contact and hosted URL are pending Apple-side registration setup.

## Change Notice

This draft must be reviewed before publication. If data practices change, update this policy and App Store Connect privacy responses before submitting a new build.
